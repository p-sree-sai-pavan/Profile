CVM++ — Stack-Based VM & Compiler

Timeline: Jan 2026 – Apr 2026
Project Type: Team Project (IITG Coding Club)
Languages: C++17 (No dependencies)
Source Code: GitHub Repo

Technical Deep Dive

CVM++ is a custom stack-based compilation and execution pipeline built from the ground up in modern C++17.

  • Lexer & Parser: Implemented a recursive-descent parser that parses algebraic expressions, loops, variables, conditional statements, and console I/O into an Abstract Syntax Tree (AST).
  • Compiler: Emits a flat uint8_t stream bytecode containing 23 custom opcodes (e.g. LOAD, STORE, ADD, JMP, HALT).
  • Forward-Jump Patching: Employs a two-pass assembler mechanism to resolve forward jump instruction offsets (e.g., in `if-else` and `while` structures) only after body bytecode emission is finished.
  • Virtual Machine: Executes bytecode via a high-performance fetch-decode-execute loop with zero external overhead. Fast O(1) slot mapping for variable lookup instead of costly string maps.
  • Interactive Disassembler: The system features a --debug flag that prints the parsed AST graphically and outputs the annotated bytecode disassembly side-by-side with instruction offset hex values.

Parker — AI Assistant with Persistent Memory

Timeline: 2025 – Present
Project Type: Individual Project
Tech Stack: LangGraph, pgvector, PostgreSQL, Python, Whisper API
GitHub: GitHub Repo

Technical Deep Dive

Parker is a private, cross-platform Telegram bot AI companion built with a focus on human-like long-term memory retrieval and semantic correlation.

  • 4-Layer Memory Hierarchy: Segmented into active context, ranked facts, multi-session project trackers, and a 90–365 day archival archive.
  • Episodic Summarization: Condenses interactions into a tree schema (chat log → day → week → month → year) to manage token contexts for LLMs efficiently over time.
  • Semantic Search: Stores 1,024-dimensional pgvector embeddings in a local PostgreSQL instance, triggering vector cosine similarity searches on every user input.
  • LangGraph Orchestration: Combines prompt engines, vector stores, memory consolidators, and chatbot handlers using a state machine layout designed to execute with 4 isolated API keys for parallel fault tolerance.
  • Deployment: Integrates custom telegram listener libraries with Whisper voice inputs to convert sound snippets into text dynamically in Python.

MWIS Solver — Maximum Weight Independent Set

Timeline: Jan 2025 – Apr 2025
Project Type: Team Project (IITG Coding Club)
Languages: C++
GitHub: GitHub Repo

Technical Deep Dive

A high-performance algorithmic solver for computing the Maximum Weight Independent Set (an NP-hard problem) on large-scale sparse and dense graphs.

  • Branch-and-Bound DFS: Traverses search space with highly pruned search trees utilizing suffix-sum bounding strategies based on degree heuristics.
  • Bitmask Conflict Check: Stores local adjacency structures using 32-bit integer bitmasks, achieving O(1) bitwise intersection calculations for conflict checks on graphs holding up to 200,000 nodes and 200 Million edges.
  • Buffered I/O speed: Swaps default std::cin with custom low-level getchar_unlocked() parsers, achieving up to 100x faster file reading during benchmarks.
  • ADD/SWAP Metaheuristic: Applies a local search neighborhood restart strategy that jumps local optima to achieve 10-50% higher weight sets within a tight 300-second execution budget.
